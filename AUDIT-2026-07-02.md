## 6. World-class options (ranked by stability-per-effort)

1. **Pre-deploy verification script (`predeploy_check.py`)** — one command that runs before every `wrangler deploy`: every page ends `</html>` + loads `site.v2.js`, single cache-bust version census, sitemap ↔ folder diff, internal link check (offline version of linkcheck), consent checkbox fingerprint on `/quotes/`, no NULs, forms point at the right workers. ~1 hour to build from the checks used in this audit; turns "audit day" findings into "can't ever ship it broken." **Biggest single stability win available.**
2. **Git + staged deploys** — already on the roadmap (§12). Version history, diffable deploys, rollback, and it kills the whole class of "which copy is truth" problems (including the sandbox-mount confusion). Effort: half a day. Do it after the current multi-project audit wave settles.
3. **Lead-pipeline alerting** — a scheduled Worker cron that test-POSTs a synthetic lead weekly and emails if the pipeline (Turnstile → worker → Resend) breaks, plus a failure digest on real send-errors. Quotes are the site's entire job; today a silent failure is only caught by noticing the inbox went quiet.
4. **Uptime + form monitoring** (external, e.g. free UptimeRobot tier on `/` + `/quotes/`) — catches Cloudflare/DNS-level outages the cron can't see itself.
5. **Turnstile fail-closed for full submissions** (al3-worker; keep partial-lead exemption) — with alerting from #3 in place, fail-open's rationale (never lose a lead) is preserved while a missing secret gets noticed. Coordinate with the worker-side audit.
6. **Consolidate blog markdown sources** (§12 open item: 23 of 38 posts have markdown) — finishing this makes `build.py` fully authoritative and the site regenerable from source, which is what makes Git migration (#2) clean.
7. **Real photography + entity SEO polish** — replace stock heroes with office/team photos, matching GBP images; modest AEO/local-SEO lift, do opportunistically.
